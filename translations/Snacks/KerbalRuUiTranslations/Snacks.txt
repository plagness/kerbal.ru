# Snacks (Snacks! Continued — жизнеобеспечение, WildBlueIndustries/Snacks) — UI-словарь,
# извлечён из скомпилированного SnacksUtils.dll (единственный *.dll в Plugin/;
# также лежат .pdb/.mdb — символы, не код). Метод: прямой разбор #US-хипа
# (PE-метаданные, ECMA-335 compressed-length UTF-16LE), НЕ monodis --userstrings
# напрямую: monodis дробит длинные UTF-16-строки на bytearray-куски и склеивает
# соседние записи хипа, из-за чего оконный текст («Snack Time!», «Stop simulators…»)
# читается фрагментами. Прямой парсер вернул 434 цельные строки байт-в-байт.
#
# KerbalRuUiTranslator.cs: Harmony-перехват отрисовки текста (GUI/GUILayout,
# GUIContent, UnityEngine.UI.Text, TMPro TMP_Text). Матч — ТОЧНАЯ строка целиком
# (Ordinal), промах = строка идёт как есть (no-op). Каждый ключ ниже сверен на
# байт-в-байт присутствие в #US-хипе.
#
# Что переведено (только СТАТИЧНЫЕ цельные строки, отображаемые целиком):
#  - Окно «Vessel Status» / Stresstimator: заголовок, кнопки (Rerun Simulator,
#    Show/Available Crew, Open/Close Stresstimator, Stop simulators…), сообщения
#    об отсутствии экипажа/аппаратов, подсказки Exempt Kerbals, длинный help-текст.
#  - Инфо-панель детали в редакторе (GetInfo): Inputs/Outputs, Note, Resource Options.
#  - Статусы преобразователя/переработчика в PAW (A-OK, Running, Vessel must be in
#    water, Requires…) и статусы состояния (Stressed Out, On Strike).
#  - Итоги производства (Critical Success/Failure, Production yield…, Duration:…).
#  - Тултип «Drag to resize the window», guiName «Max Recycling».
#
# Пропущено НАМЕРЕННО (движок не сматчит цельную строку — останется как есть):
#  - Строки с токенами формата: {0:n2}, <<percent>>/<<amount>>/<<KerbalName>>,
#    #autoLOC_* (сток-ключи) — в UI приходят после String.Format/подстановки.
#  - Строки-ФРАГМЕНТЫ, склеиваемые в рантайме с именами/числами: «<color=white>Crew: »,
#    « - Status: », «Needs more », «Progress: », письма «Subject: …»/«From: » и т.п.
#    Собранный целиком лейбл содержит динамику → точного совпадения нет.
#  - Сообщения о смерти/обмороке от нехватки снеков: постятся как «<имя> has died…»
#    → цельная строка включает имя кербонавта, ключ без имени не совпадёт.
#  - Многострочные (реальные \n/\r\n) — построчный формат словаря их не хранит.
#  - Обобщённые «Fail»/«Success»/«Snacks»/«Soil»/«None»/«N/A» — имена ресурсов/значения
#    (идут через локализацию ресурсов) либо слишком общие (высокий риск ложного матча).
#  - guiName деталей и заголовки настроек сложности лежат в custom-attributes/#Blob,
#    а не в #US — этим дампом не покрываются (нужен отдельный проход по атрибутам).
#
# Формат: Английский<TAB>Русский. Разделитель — первый TAB. # — комментарий.
# Ведущие/хвостовые пробелы и пунктуация ключей сохранены байт-в-байт.
