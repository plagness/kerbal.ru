# B9_Aerospace_ProceduralWings — wing/control-surface shape editor
# (chord/thickness/sweep/edges, the "press J to edit" window).
#
# MAJOR FINDING: this mod already ships a FULL, complete Russian localization
# via KSP's stock #autoLOC_ system:
#   GameData/B9_Aerospace_ProceduralWings/Localization/ru.cfg (171 keys)
# matched 1:1 against the English source in Localization/refactor.cfg. It
# covers literally the entire shape editor: every field label, every dropdown
# value (Rounded/Biconvex/Triangular/Uniform/Standard/Reinforced...), every
# tooltip paragraph, and the 3 part title/description blocks. This looks like
# prior work from this same kerbal.ru project (ru.cfg's mtime is newer than
# every other file the mod ships, and its phrasing matches this project's
# established RU terminology).
#
# The 116 pairs below are extracted directly from refactor.cfg/ru.cfg (same
# approach as ConformalDecals.txt) so the SAME Harmony dictionary also covers
# this mod if the game runs in English locale — Text.text still ends up
# holding the English string in that case, which is exactly what this
# framework matches on. Skipped only the 4 entries where English == Russian
# already (a literal "#", the separator line, and the material names
# LRSI/HRSI which stay as acronyms in Russian too).
#
# GAP FOUND BY DIRECT ldstr INSPECTION: a handful of tooltip/label strings are
# NOT in ru.cfg at all — they're literal English fallback branches inside
# WingProcedural::UpdateTooltipText() / DrawCheck() calls for tooltip ids
# 105-107, 301 (0x69-0x6b, 0x12d) that sit right next to sibling ids which DO
# route through Localizer.Format("#autoLOC_..."). Looks like these were added
# in a mod update after the last "make localization" pass and never
# backported to ru.cfg. Also found: two custom difficulty-settings page
# titles ("Controls"/"Log Settings", WPSensitivity/WPDebug.get_Title()) and
# a crash-tolerance difficulty field, an "Invalid" tank-setup fallback label,
# and two reflection-fallback strings (TitleNotAvailable/VersionNotAvailable)
# shown only if the assembly's own version metadata is missing. These 13
# entries are new translations (not sourced from ru.cfg) — worth a follow-up
# PR to add them to B9's own ru.cfg too, since that's the more correct fix.
#
# Skipped as non-UI (confirmed by tracing call sites): hundreds of
# Debug.Log-only strings, FAR/NEAR integration diagnostics, shader/material
# property names (_MainTex, sharedMaterialEL...), reflection field names
# (ctrlHandleLength1, handleWidthRootBack...), GameObject.Find() lookups
# ("Normal", "CtrlSurf"), format specifiers (F2/F3/G5), and the
# install-path-check popup's long error paragraphs (not part of the shape
# editor itself). "OK" skipped — same in Russian UI convention, no-op.
