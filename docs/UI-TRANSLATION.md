# Русификация интерфейсов модов (UI, не детали)

Все переводы до этого момента — данные (`Localization{}`-ключи в `.cfg`). Интерфейсы модов
(окна MechJeb2, панели Kerbalism, окно FAR и т.д.) — это **текст, зашитый прямо в скомпилированный
C#**, а не в `Localization`-ключи. Ни один из проверенных модов не использует `Localizer.Format`/`#LOC_`
для своего UI (проверено `strings`-дампом DLL всех крупных кандидатов — 0 совпадений везде).

## Подход

Единственный вменяемый способ без форка/перекомпиляции чужих модов — **Harmony-патч на уровне
самого Unity IMGUI/UGUI**, а не на уровне каждого мода отдельно:

- Перехватываются точки входа отрисовки текста: `GUI.Label/Button/Box/Toggle`,
  `GUILayout.Label/Button/Box/Toggle`, все 4 конструктора `GUIContent(string, ...)`,
  `UnityEngine.UI.Text.text` (сеттер).
- При каждом вызове входящая строка ищется в словаре `английский → русский` (точное совпадение).
  Нашли — подменяем. Не нашли — строка идёт как есть, без изменений.
- Словарь — внешние `.txt`-файлы (`Английский<TAB>Русский` построчно), не часть кода. Расширение
  перевода = правка текстового файла, **не пересборка DLL**.

### Почему это безопасно

- Патч не модифицирует ни один файл мода — работает поверх, как отдельный аддон.
- Несовпадение = отсутствие изменений, никогда не «ломает» текст: даже если фраза одного мода
  случайно совпадёт с фразой другого — подменится одинаково корректно (это унифицирует терминологию,
  а не портит её).
- Строки, отформатированные с живыми числами (`"Высота: 1234.5 м"`) естественным образом никогда
  не совпадут с точным ключом словаря — просто останутся непереведёнными, без побочных эффектов.
- Перехватываются **только функции отрисовки текста**, не логика/сравнение строк/ключи сохранений —
  риска задеть игровую механику нет.

### Проверено перед сборкой

Сигнатуры всех патчей сверены с `monodis`-дампом реальных `UnityEngine.IMGUIModule.dll` /
`UnityEngine.UI.dll` из установленной игры — 100% совпадение (не предположение, а факт).

## Реализация

- `tools/ui-translator/src/KerbalRuUiTranslator.cs` — исходник, собирается `mcs` (Mono) в
  `GameData/kerbalru-ui-translator/Plugins/KerbalRuUiTranslator.dll`. Пересборка: `tools/ui-translator/build.sh`.
- `GameData/kerbalru-ui-translator/KerbalRuUiTranslations/<Mod>.txt` — словари по одному на мод.
- Зависимость: `000_Harmony` (уже в сборке).

## Область охвата (v1, 2026-07-23)

Извлечён и переведён текст из 24 модов с собственным интерфейсом — **1926 строк** (1867 уникальных
ключей). По объёму: MechJeb2 (224), Waterfall (204), AtmosphereAutopilot (162), RealChute (162),
RealAntennas (147), Scatterer (160), Skopos (94), ProceduralParts (63), RealFuels (54), FAR (53),
KSPCommunityFixes (57), остальные — от 2 до 126 строк каждый.

**Уже полностью локализованы нативно (Localizer/`#LOC_`), Harmony-словарь не нужен или дополняет
малый остаток**: MechJeb2 (свой `ru.cfg`, 1440 строк, переводчик craftersmine — наш словарь
покрывает только ~224 нехардкоженные строки), KerbalChangelog (0 строк — целиком через Localizer,
уже есть `ru.cfg` в этом репозитории), PatchManager (7 строк — почти всё уже через `Lang/ru.cfg`),
ConformalDecals и B9_Aerospace_ProceduralWings (текст уже в родных `ru.cfg`, наш словарь дублирует
те же пары на случай английского языка игры).

**Не найдены/не установлены физически в этой сборке** (только осиротевший файл перевода партов/квестов,
плагина нет): `KerbalEngineer`, `Trajectories`, `Kerbalism`, `EditorExtensionsRedux` — переводить их
UI нечего, пока мод не доустановлен.

## Проверка перед релизом (2026-07-23)

Полный словарь прогнан через состязательную ревизию (7 независимых рецензентов вычитали каждую
строку целиком + отдельная верификация каждой находки другим агентом с нуля):

- **42 подтверждённых бага исправлено**: согласование рода аббревиатуры ЭДСУ (женский род —
  «Стандартная», не «Стандартное»), «ложные друзья переводчика» (`authority` → «эффективность
  управления», не «авторитет»; `strength` → «интенсивность», не «прочность» — конфликтовало с
  «director strength» в том же файле), несогласованность термина vessel (аппарат/корабль) внутри
  одного файла, несколько случаев потерянного завершающего пробела перед местом конкатенации
  динамического значения (`"Selected part: "` и подобные — обрезание даст слипшийся текст
  `"Выбранная деталь:ROCapsule"` вместо `"Выбранная деталь: ROCapsule"`).
- **53 находки помечены как низкий риск** (слишком общее слово без контекста типа "Name"/"OK",
  не критично) — не исправлялись, стоит перепроверить визуально при первом живом тесте.
- **1 межмодовый конфликт терминов принят как есть** (не считается ошибкой, см.
  `KNOWN_CROSS_MOD_AMBIGUITIES` в `validate_dict.py`): `"Bottom"` в MechJeb2 — кнопка «переместить
  в конец списка», в ProceduralParts — направление «низ» детали. Один и тот же английский ключ,
  разный смысл в разных модах; плоский общий словарь физически не может передать оба значения
  одновременно — какой файл загрузится позже, тот и победит в конкретной сессии.
- **Найден и исправлен настоящий блокер во фреймворке**: `LoadFile()` не была обёрнута в try/catch
  индивидуально — один нечитаемый `.txt` (гонка ФС, блокировка антивирусом, права доступа в
  контейнере на Steam Deck) мог тихо отключить весь Harmony-патчинг на всю сессию (`KSPAddon.Instantly`
  не даёт повторной попытки). Исправлено: загрузка словаря и `Harmony.PatchAll` — независимые
  try/catch, плюс try/catch на уровне каждого отдельного файла.
- Отдельно проверено через изолированный тестовый харнесс (`tools/ui-translator/src/TestHarness/`) —
  все 23 точки перехвата реально резолвятся через `Harmony.PatchAll` против настоящих
  `UnityEngine.IMGUIModule.dll`/`UnityEngine.UI.dll` из установленной игры, без единой ошибки.

## Известные ограничения

- Динамический текст с постоянно меняющимися числами не переводится принципиально (см. выше) —
  переводятся только статичные подписи, заголовки, кнопки, тултипы.
- Несколько строк с буквальным переносом строки внутри значения (не место для TAB-разделённого
  формата) пропущены явно, с пояснением в шапке соответствующего файла.
- `GUIContent.text`-сеттер (в отличие от 4 конструкторов) не патчится — мод, кэширующий
  `GUIContent` и переприсваивающий `.text` напрямую, обойдёт перевод на этом конкретном виджете.
- Если какой-то мод сравнивает виджет-текст с английским литералом после того, как текст уже
  подменён патчем (`label.text == "Locked"`) — сравнение молча сломается. Не ловится статическим
  анализом, только функциональной проверкой во время живого теста.
- Первый живой запуск с этим патчем — **первая проверка новой техники в этом проекте**: раньше
  все изменения были декларативными `.cfg`-данными, это первый раз, когда правится поведение через
  скомпилированный код. Нужна внимательная проверка (не только текста, но и что игра вообще не упала).
- Словарь можно спокойно пополнять позже без пересборки — просто добавляя строки в `.txt`-файлы.
- `tools/ui-translator/validate_dict.py` подключён в CI (`.github/workflows/project-checks.yml`).
