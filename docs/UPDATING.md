# Обновление и управление сборкой kerbal.ru

`install.sh` — **менеджер сборок**: он находит уже установленную сборку по
`<KSP>/.kerbalru-build.json` и умеет её обновить, сменить на другую или обновить только
переводы. Отдельного «установщика русификатора» больше нет — язык и переводы накатываются
тем же скриптом.

Windows-установщик (`install.ps1`) пока **не существует** — планируется. Сейчас используйте
`install.sh` (Linux, macOS, Steam Deck).

## Обычное обновление

Закрой KSP и снова запусти установщик — интерактивно или флагом `--update`.

```bash
# интерактивное меню (определит текущую сборку и предложит действие)
curl -fsSL https://kerbal.ru/install.sh | bash

# без интерактива — обновить текущую сборку
curl -fsSL https://kerbal.ru/install.sh | bash -s -- --update
```

`--update` заново ставит `core`+`recommended` текущей сборки через CKAN, накатывает свежие
переводы под реально установленные моды и перезаписывает состояние в `.kerbalru-build.json`.

## Флаги

| Флаг | Что делает |
|---|---|
| (без флага) | сборка стоит — предложит обновить / сменить / только переводы; иначе — выбор из каталога |
| `--build <id>` | поставить или сменить сборку на конкретную (`operator`, `rp1`, …) |
| `--update` | обновить текущую установленную сборку |
| `--ru-only` | обновить только переводы поверх текущей сборки (моды и CKAN не трогаются) |
| `--list` | показать каталог сборок и выйти |
| `--yes` / `-y` | не спрашивать подтверждений (для CI/скриптов) |

Первым позиционным аргументом можно передать путь к KSP, если автопоиск не нашёл игру:
`install.sh "/путь/к/Kerbal Space Program" --update`.

## Обновить только переводы

Если моды не менялись, а нужен свежий русский текст:

```bash
curl -fsSL https://kerbal.ru/install.sh | bash -s -- --ru-only
```

`--ru-only` выставляет `LANGUAGE = ru`, кладёт движок ui-translator в
`GameData/kerbalru-ui-translator/Plugins/`, ui-словари — в
`GameData/kerbalru-ui-translator/KerbalRuUiTranslations/`, а Localization-патчи —
в `GameData/zzz-kerbalru-translations/<mod>/` (префикс `zzz-` гарантирует применение после
Realism Overhaul). Переводы кладутся **только под реально установленные моды** — перевод
отсутствующего мода просто пропускается.

## Сменить сборку

```bash
curl -fsSL https://kerbal.ru/install.sh | bash -s -- --build operator
```

При смене на другую сборку установщик удаляет моды текущей сборки (стоковый `Squad`/
`SquadExpansion` и папка `saves/` остаются) и ставит новую. Сохранения не удаляются, но могут
не загрузиться в другой сборке — перед сменой имеет смысл отдельно сохранить `saves/`.

## Что хранится в каталоге KSP

| Путь | Назначение |
|---|---|
| `.kerbalru-build.json` | состояние установки: id сборки, имя, версия KSP, система, список установленных модов |
| `settings.cfg.bak-kerbalru` | исходный `settings.cfg` до первого переключения языка на `ru` |

Установщик никогда не очищает `GameData` целиком при обновлении: `--update`/`--ru-only`
переустанавливают свои файлы поверх, а полная очистка до стока происходит только при явной смене
сборки (`--build <id>` на другой id).

## После обновления модов через CKAN

Если ты обновлял моды сборки вручную через CKAN, порядок такой:

1. обновить моды и зависимости через CKAN;
2. запустить KSP один раз, если обновлялся ModuleManager или состав модов;
3. повторно запустить `install.sh --ru-only`, чтобы вернуть переводы под обновлённые моды;
4. запустить KSP и при необходимости пересобрать аудит кэша.

Если новый релиз мода переименовал ключи или детали, старый перевод может стать неполным даже
при актуальной библиотеке переводов. Сообщи об этом через шаблон «Проблема перевода» и укажи
версию мода.

## Каталог сборок и релизы

Актуальный состав каталога — `builds/_catalog.json` (генерируется `tools/gen_catalog.py`),
библиотека переводов — `translations/_index.json`. В curl-режиме `install.sh` тянет каталог и
переводы из ветки `main`. Стабильные снимки помечаются тегами вида `vГОД.НОМЕР`
(`v26.1` — первый релиз 2026 года), см. [CHANGELOG.md](../CHANGELOG.md) и
[GitHub Releases](https://github.com/plagness/kerbal.ru/releases).

## Для maintainer: номер релиза

Для нового релиза maintainer запускает `python3 tools/bump_release.py`: команда увеличивает номер
в пределах текущего года и обновляет `data/project.json`, `VERSION`, `CITATION.cff`. Явный номер
можно передать как `python3 tools/bump_release.py v26.2`. Полная процедура релиза —
в [MAINTAINING.md](MAINTAINING.md#8-релиз-и-обновление).
