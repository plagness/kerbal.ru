# Покрытие переводом — реальные цифры (проверено по игровому кэшу)

⚠️ **Обновлено 2026-07-22.** Раздел 0 ниже — цифры, проверенные по `ModuleManager.ConfigCache` (то, что реально загружает игра), а не по факту наличия файла локализации. Более ранняя методика («файл `ru.cfg` есть → мод переведён», разделы 1-3 ниже) **переоценивала готовность**: файл может лежать в папке и даже содержать правильный ключ, а игра всё равно показывает английский. Разделы 1-3 сохранены как raw-данные по наличию файлов, но не считайте их показателем реальной русификации — доверяйте разделу 0.

### Известная (пока не решённая) причина части поломок

На примере SXT, деталь `SXTSputnik`: ключ `#LOC.SXT_SXTSputnik_title` определён и в оригинале мода (`Localization/en-us.cfg`), и в нашем `Localization/ru.cfg` — причём в `ru.cfg` лежит **другой, переведённый текст**, не копия английского. Тем не менее игра показывает `Sputnik PS-1` — ни английский из файла, ни наш русский вариант. Проверено и исключено: BOM/кодировка (все файлы честный UTF-8 без BOM), дублирующиеся блоки `ru`/`en-us` (по одному на файл). Причина пока не найдена.

**Практический вывод:** пока причина не ясна, повторный перевод SXT / ROCapsules / VenStockRevamp / RealChute тем же методом рискует воспроизвести ту же невидимую поломку и потратить работу впустую. Приоритет — сначала диагностика этого механизма, потом уже добивание перевода. См. [ROADMAP.md](ROADMAP.md).

### Оценка объёма оставшейся работы

Считаем по `title` — самому заметному полю, тому, что игрок видит в списке деталей VAB/SPH:

| Мод | Осталось title (детали) |
|---|---:|
| SXT | 199 |
| ROEngines | ~215 (часть — легитимные designator-исключения, реально меньше) |
| VenStockRevamp | 76 |
| ROCapsules | 94 |
| ROTanks | 12 |
| ROHeatshields | 6 |
| ProceduralParts | 5 |
| RealChute | 4 |
| ProceduralFairings | 3 (новая находка: `KzThrustPlate`, не был в списке известных исключений) |
| **Итого** | **≈ 610–650** |

Плюс сопоставимое количество строк `description` (около 470 недостающих) и `manufacturer` (около 965, но это самое низкоприоритетное поле — редко на виду).

**Итого по объёму:** порядка 600-650 title + аналогичный порядок description ≈ **1200-1500 строк**, сопоставимо по масштабу с изначальным проходом этого проекта (~3500 строк за один ~2-часовой автономный прогон с параллельными агентами). Технически это отдельный проход того же порядка трудоёмкости — но начинать его стоит только после диагностики причины выше, иначе есть шанс просто повторить невидимую поломку.
