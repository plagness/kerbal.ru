# Покрытие переводом — реальные цифры (проверено по игровому кэшу)

⚠️ **Обновлено 2026-07-22.** Раздел 0 ниже — цифры, проверенные по `ModuleManager.ConfigCache` (то, что реально загружает игра), а не по факту наличия файла локализации. Более ранняя методика («файл `ru.cfg` есть → мод переведён», разделы 1-3 ниже) **переоценивала готовность**: файл может лежать в папке и даже содержать правильный ключ, а игра всё равно показывает английский. Разделы 1-3 сохранены как raw-данные по наличию файлов, но не считайте их показателем реальной русификации — доверяйте разделу 0.

## 0. Реальный охват по деталям (партам) — проверено по кэшу

Метод: для каждого из 20 модов взят список реальных `name` деталей из исходников мода, каждое имя сверено с полями `title` / `description` / `manufacturer` в финальном `GameData/ModuleManager.ConfigCache` — это уже полностью пропатченный конфиг, который игра использует напрямую. Наличие кириллицы = деталь реально на русском в игре; отсутствие = не работает независимо от того, что лежит в файлах.

### ✅ Исправление применено 2026-07-22 (структурно проверено, ждёт подтверждения кэшем)

Для 6 худших модов (SXT, ROCapsules, VenStockRevamp, RealChute, ProceduralParts, ProceduralFairings) написан и применён **новый** перевод — **не через родную систему локализации мода** (там необъяснённый сбой, см. ниже), а тем же способом, что уже даёт 98-100% у ModularLaunchPads/ROSolar: свой MM-патч `%title = #LOC_kerbalru_<mod>_<part>_title` со свежим неймспейсом, который не может пересечься ни с чем существующим. Английский текст-источник взят из реального `ModuleManager.ConfigCache` (то, что игра показывает прямо сейчас), а не из протухших комментариев в исходниках — так лучше во всех случаях.

### Причина исходной поломки (обойдена, не решена)

На примере SXT, деталь `SXTSputnik`: ключ `#LOC.SXT_SXTSputnik_title` определён и в оригинале мода (`Localization/en-us.cfg`), и в нашем старом `Localization/ru.cfg` — причём там лежал **другой, переведённый текст**, не копия английского. Тем не менее игра показывала `Sputnik PS-1` — ни английский из файла, ни русский вариант. Проверено и исключено: BOM/кодировка (все файлы честный UTF-8 без BOM), дублирующиеся блоки `ru`/`en-us` (по одному на файл). Причина так и не найдена — вместо дальнейшей диагностики выбрали обходной путь (см. блок исправления выше): свой патч с чистым неймспейсом вместо попытки почини́ть родной механизм мода. Если кто-то захочет всё же разобраться в первопричине — старый (нерабочий для части ключей) `ru.cfg` каждого из этих 4 модов остался на месте нетронутым, для сравнения.
