# Управление проектом

kerbal.ru развивается публично и прагматично: обсуждение открыто для всех, а ответственность за выпуск и безопасность остаётся у maintainer.

## Роли

- **Участник** — сообщает об ошибках, обсуждает решения, тестирует или отправляет PR.
- **Постоянный участник** — регулярно помогает в конкретной области и может быть приглашён к triage/review.
- **Maintainer** — принимает изменения, управляет релизами, инфраструктурой и модерацией.

Роль определяется вкладом и доверием, а не количеством сообщений. Сейчас окончательное решение и доступы к публикации принадлежат владельцу репозитория [@plagness](https://github.com/plagness).

## Как принимаются решения

1. Небольшие исправления принимаются через обычное ревью PR.
2. Новые поддерживаемые моды, изменение формата релиза и потенциально ломающие изменения сначала обсуждаются в issue или Discussion.
3. Предпочтение получает решение, которое проще проверить в реальной игре, безопаснее обновить и легче сопровождать.
4. Если консенсуса нет, maintainer фиксирует решение и аргументы публично.

## Релизы

- `main` — интеграционная ветка и источник сайта; она может содержать ещё не выпущенные изменения переводов.
- теги `vYY.N` — неизменяемые стабильные версии русификатора: `YY` — две цифры календарного года, `N` — порядковый номер выпуска в этом году;
- `v26.1` означает первый выпуск 2026 года, следующий — `v26.2`, а в новом году счётчик начинается заново (`v27.1`);
- несовместимые изменения отдельно и заметно описываются в release notes и документации обновления, а не кодируются скрытой семантикой номера.

Релизный PR обновляет `data/project.json`, пользовательскую документацию и при необходимости миграцию. Скрипт синхронизации получает из JSON `VERSION`, README и `CITATION.cff`. Тег должен совпадать с `VERSION`; GitHub Actions создаёт Release. Установщики по умолчанию используют последний стабильный Release, а канал `main` предназначен для тестирования.

## Изменение этого документа

Governance меняется обычным PR с публичным объяснением причины. Существенные изменения ролей или правил релиза предварительно обсуждаются.
