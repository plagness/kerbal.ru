# Профиль производительности kerbal.ru (`install.sh --perf`)

> **Legacy (сборка `rp1`).** Материалы этого каталога — declutter/оптимизация старой сборки
> RO/RSS/RP-1 (флаг `--perf` в текущем `install.sh` не реализован). При новом концепте (хаб +
> каталог сборок) это **низкий приоритет**; оставлено как справка по перф-профилям RSS-сборки.

Опциональный, **по умолчанию выключенный** набор правок, ускоряющий сборку RO/RSS/RP-1 на
KSP 1.12.5. Включается одной командой. Полное обоснование и таблицы значений — в
[`docs/PERFORMANCE.md`](../docs/PERFORMANCE.md); здесь — практическая выжимка: что делает
профиль, три канала доставки, три профиля-цели, как откатить и что настроить руками на Deck.

Принцип (решение оператора): профиль **адаптивный** под железо; **при неуверенности —
консервативно** (низкие настройки, чтобы точно не стало хуже). Правки настроек субъективны и
зависят от железа — поэтому opt-in, а не по умолчанию (в духе `docs/FOR-AGENTS.md`: не
оптимизировать `settings.cfg` вслепую).

## Как включить / откатить

```bash
install.sh [путь] [--full|--ru-only] [--perf[=auto|mac|deck|generic|off]]
```

| Флаг | Что делает |
|------|-----------|
| (без `--perf`) | как сейчас, ничего перф-специфичного (opt-in) |
| `--perf` / `--perf=auto` | определить платформу и применить её профиль |
| `--perf=mac` | форсировать Mac-профиль |
| `--perf=deck` | форсировать Deck-профиль |
| `--perf=generic` | консервативно-низкий профиль для неизвестного Linux/Win десктопа |
| `--perf=off` | **откат**: вернуть `settings.cfg.bak-kerbalru`, удалить `GameData/kerbalru-perf/`, (опц.) вернуть 8K-текстуры через CKAN |

Откат безопасен: канал C восстанавливается из бэкапа `settings.cfg.bak-kerbalru` (его уже
создаёт установщик), канал B — удалением нашего config-мода. Канал A (текстуры) откатывается
обратным CKAN-свапом при желании — не обязателен.

## Три профиля-цели

| Профиль | Железо | Стратегия |
|---------|--------|-----------|
| **mac** | M1/Rosetta, GPU через старый Apple GL | режем GPU агрессивно (тени/Scatterer), текстуры 8K→**4K**, физика стабильнее |
| **deck** | слабый RDNA2 APU, 16 ГБ shared | режем сильнее всего: текстуры 8K→**2K** (обход бага DXVK #5140), океан off, объёмные облака вырезаны, тени 1/0 |
| **generic** | неизвестный Linux/Win десктоп | консервативно-низкий, безопасный: текстуры 8K→**2K**, значения как у deck (без Deck-специфики питания) |

Детект платформы (§6.5 PERFORMANCE.md): `uname -s` = Darwin → mac; Linux + признаки SteamOS
(`/etc/os-release` `ID=steamos`, путь `/run/media/deck`, `steamos-session-select`) → deck;
прочий Linux → generic (консервативный) либо явный `--perf=mac|deck`.

## Три канала доставки

| Канал | Что доставляет | Кто применяет | Файлы в репозитории |
|-------|----------------|---------------|---------------------|
| **A. CKAN-свапы** | RSS-текстуры 8K→4K/2K; анти-фриз мод (HeapPadder); опц. снятие SXT/VenStock | тот же headless-CKAN, что уже в `install.sh` | [`ckan-swaps.txt`](ckan-swaps.txt) + [`heappadder/padheap.cfg`](heappadder/padheap.cfg) |
| **B. Наш config-мод** | MM-патчи Scatterer/EVE + KSPCF-extras (значения по профилю) | копируется в `GameData/kerbalru-perf/` из репо | `gamedata/kerbalru-perf-<profile>/kerbalru-perf.cfg` |
| **C. Фрагмент settings.cfg** | стоковые графика/физика (тени, физ-шаг, лимиты FPS) | merge в `settings.cfg` по ключам, до запуска игры | `settings/common.env` + `settings/<profile>.env` |

**Легальность:** чужие моды (текстуры, HeapPadder) мы НЕ храним — их ставит официальный CKAN
(канал A). Наши конфиги (каналы B, C и `padheap.cfg`) распространять можно — они наши.

### Что в каком канале (детально)

| Изменение | Канал | mac | deck | generic |
|-----------|:---:|:---:|:---:|:---:|
| RSS-текстуры (провайдер-свап) | A | 4K | 2K | 2K |
| Анти-фриз HeapPadder + `padheap.cfg` (512 МБ) | A | ✅ | ✅ | ✅ |
| Снятие SXTContinued / VenStockRevamp (опц., за флагом) | A | ○ | ○ | ○ |
| Scatterer: тени 8192→2048/1024, океан, FFT-сетка | B | режем | режем сильно | режем сильно |
| EVE: объёмные облака `layerVolume` | B | оставить | вырезать | вырезать |
| KSPCF: `FewerSaves`, `SkipIndentsOnSaves…` = true | B | ✅ | ✅ | ✅ |
| `SHADOWS_QUALITY` 4 → 2 / 1 | C | 2 | 1 | 1 |
| `PHYSICS_FRAME_DT_LIMIT` 0.04 → 0.03 | C | ✅ | ✅ | ✅ |
| `FRAMERATE_LIMIT`, `LIGHT_QUALITY`, отражения и пр. | C | mac-набор | deck-набор | deck-набор |

✅ применяется · ○ опционально (нужен явный opt-in) · «режем» — значения см. PERFORMANCE.md §2.4/§2.5

## Анти-фриз: HeapPadder + наш `padheap.cfg`

По умолчанию — **HeapPadder** (пэддит Mono-heap автоматически на загрузке, без хоткея →
подходит для автоустановки). Наш [`heappadder/padheap.cfg`](heappadder/padheap.cfg) урезает
пэддинг до **512 МБ** (родной авто-пресет HeapPadder на 16 ГБ ставит 4096 МБ — на нашей
тяжёлой RO/RSS-сборке с общей памятью это своп). Кладётся **после** CKAN-install мода, **до**
первого запуска игры, в `GameData/HeapPadder/PluginData/padheap.cfg`.

> **Грабля (для оператора):** в CKAN HeapPadder помечен несовместимым с 1.12.5
> (`ksp_version_max = 1.7` — застой метаданных, не реальная поломка). Ставится флагом
> `--allow-incompatible`. Мод — чистый C#, на 1.12.5 работает. Если этот флаг нежелателен,
> в `ckan-swaps.txt` есть готовая альтернатива **MemGraph** (совместим, но пэддит по хоткею
> `Mod+End` — хуже для автоматики). Детали и id — в `ckan-swaps.txt`.

## Steam Deck — чеклист вручную (НЕ автоматизируется)

Эти вещи живут в Game Mode Deck, а не в файлах игры — установщик их не трогает
(`docs/PERFORMANCE.md` §5.2). Сделай в Quick Access → Performance для профиля игры:

- [ ] **Кап FPS = 30** (LCD 60 Гц) или 30/40 (OLED) — KSP в сложных сценах всё равно не выдаст больше; кап убирает judder и жор батареи.
- [ ] **TDP = 15 Вт** (максимум) — игре нужен CPU-запас.
- [ ] **Manual GPU Clock низкий (~400–600 МГц)** — трюк для CPU-bound игр: GPU не отъедает общий термо/power-бюджет у CPU (после отключения визуалок высокий GPU не нужен).
- [ ] **FSR через gamescope** — рендер ~540–600p → апскейл в 800p, если оставили что-то из визуала. Half-rate shading в gamescope — ещё дешевле.
- [ ] **Ставить KSP на внутренний NVMe, НЕ на microSD** — загрузка ассетов I/O-bound; microSD в 3–4 раза медленнее.
- [ ] **Native Linux vs Proton:** ужал текстуры до 2K → **Proton (DXVK)** эффективнее на RDNA2 (overspill не срабатывает), Proton GE актуальнее стокового. Оставил крупные текстуры → native Linux безопаснее (не ловит #5140, но медленнее).

## Грабли и порядок применения (кратко)

- **Кэш ModuleManager.** Профиль ставится **одним заходом** с русификатором, чтобы MM пересобрал ConfigCache ровно раз (иначе каждый запуск = ~6 мин, см. PERFORMANCE.md §4). Первый запуск после установки ~5–6 мин (сборка кэша), последующие ~2.5 мин — это норма, не «хромает».
- **Канал C — до запуска игры.** KSP перезаписывает `settings.cfg` на выходе; патчить только при закрытой игре. Если `settings.cfg` ещё нет (свежий CKAN до первого запуска) — канал C пропускается, применится при повторном `--perf` после первого старта. Каналы A и B при этом отрабатывают.
- **Порядок:** A (CKAN-свапы) → `padheap.cfg` → B (config-мод, вместе с русификатором) → C (settings.cfg, последним, при закрытой игре).
- **Mac-гигиена (16 ГБ в обрез):** перед запуском закрыть браузеры/Electron/Docker; ноутбук на зарядке + «Высокая производительность»; внешнее охлаждение. Launch-опции `-force-glcore/-force-metal` на Mac **не** дают Metal — не карго-культить.
