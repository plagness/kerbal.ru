# AtmosphereAutopilot — UI strings extracted from AtmosphereAutopilot.dll + AtmosphereAutopilot.UI.dll
# (v1.6.1.0 / UI v1.0.0.0). Two extraction passes were needed:
#  1) `strings -a` on the raw DLL bytes (finds identifiers + AutoGuiAttr/AutoHotkeyAttr custom
#     attribute argument strings, which .NET stores as compact UTF-8 in the attribute blob heap).
#  2) A custom UTF-16LE scanner (macOS `strings` cannot see this) for ldstr literals used directly
#     in GUILayout.Label/Window calls — this is where most toast messages and window titles live.
# Skipped by design: type/method/field identifiers, ConfigNode persistence keys (snake_case),
# .pdb/.dll paths, exception messages from the internal matrix-math library, version/GUID/copyright
# metadata, string.Format templates with {0}-style placeholders (the substituted runtime text never
# equals the template, so they can never match), and strings containing an embedded literal newline
# (the loader's File.ReadAllLines splits on those before the tab-parser ever sees them, so such
# strings can never be represented as a single dictionary line).
