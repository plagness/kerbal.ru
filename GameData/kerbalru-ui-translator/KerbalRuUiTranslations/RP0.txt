# RP0 -- словарь UI-строк, извлечённых напрямую из скомпилированного RP0.dll
# (KSP-RO/RP-0 -- ядро карьеры RP-1: авионика, оснастка (tooling), программы, интеграция/сборка (KCT),
# экипаж и тренировки, доверие (Confidence), лидеры Административного здания, апгрейды).
# Формат: Английский<TAB>Русский. Несовпадение = строка остаётся как есть (см. tools/ui-translator).
#
# Извлечение: полная дизассемблировка monodis (MONO_PATH со всеми зависимостями RP-1: ROUtils,
# ContractConfigurator, ClickThroughBlocker и др. -- без них monodis обрывается на ROUtils) + разбор
# IL по каждому ldstr с симуляцией стека: строка попадает в словарь ТОЛЬКО если БЕЗ изменений доходит
# до одной из патчащихся функций -- GUILayout/GUI Label/Button/Box/Toggle/Toolbar/Window, GUIContent,
# UnityEngine.UI.Text.set_text, или до TMPro.TMP_Text.set_text через стоковые PopupDialog.SpawnPopupDialog
# / MultiOptionDialog / DialogGUILabel/Button/Toggle. Аргументная позиция проверена: у SpawnPopupDialog
# перехватываются title/message/button, но НЕ внутреннее имя диалога (arg2) и НЕ titleExtra (arg9).
# Параметры детали в PAW (guiName у [KSPField]/[KSPEvent]) -- из custom-attribute блобов, рендерятся
# через UIPartActionLabel -> TMPro (Range Safety, Disable Avionics, Contr. Mass и т.п.).
#
# СОЗНАТЕЛЬНО НЕ вошли (не пропуск, проверено по IL):
# - Строки через Localizer.Format / GetStringByTag и теги #rp0_* / #autoLOC_* -- резолвятся в англ.
#   текст из штатной локализации RP-1 (.cfg), литерал-тег до рендера не доходит; переводятся отдельно
#   на уровне cfg-локализации мода, не здесь. Текст контрактов (#cc.*) уже в kerbalru-quests-fix.cfg.
# - Аргументы string.Format/Concat/StringBuilder (динамика с числами/именами/датами -- {0}, тоннаж,
#   имена аппаратов) -- точное побайтовое совпадение невозможно.
# - Внутренние имена диалогов/локов/ключей ConfigNode, имена методов-событий, Debug.Log.
# - Одиночные символы/стрелки (X, v, ^, +, -, <<, >>) и одноимённые единицы (m, t) -- риск ложных
#   глобальных совпадений; многосимвольные единицы (km, m/s, kJ) включены.
# - 3 строки с реальным переводом строки (\n) -- загрузчик читает файл построчно (ReadAllLines) и не
#   разэкранирует \n, поэтому многострочный ключ технически непредставим: "No vessels in storage!\n...",
#   "No tech nodes are being researched!\n...", "Save as\nNew Preset" -- оставлены как есть.
# - CC_RP0.dll (логика контрактов, весь текст через #cc.*-ключи) и RP0InstallChecker.dll (единственное
#   сообщение собирается динамически) -- UI-литералов под перевод не дали (0 строк).
