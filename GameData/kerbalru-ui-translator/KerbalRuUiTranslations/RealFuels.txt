# RealFuels — UI strings extracted from RealFuels.dll: the tank-type/propellant configuration
# window (ModuleFuelTanks / RealFuelsWindow — tank type switcher, volume bar, wet/dry mass), the
# engine-config GUI with its reliability simulation charts (RFEngineConfigGUI — survival
# probability vs. burn time, failure-probability heatmap), and the Master-Status-Display-style
# column headers used by both.
#
# Method: `monodis --userstrings` for the raw string heap, then a full IL disassembly
# (`monodis --output`, KSP's Managed/ dir on MONO_PATH) to check what each candidate ldstr
# actually feeds:
#   - kept when passed directly to RealFuelsWindow.MassCell/SetTooltip, stored as its own
#     string[] array element (table/legend/tab-name arrays — e.g. the Failure Risk legend, the
#     Min%/Ignition%/Pres-Fed/0-DU/Max-DU column-header array, the R&D/Settings KSC-window tab
#     array), returned whole from a GetModuleTitle()-style override, or used as a PopupDialog
#     title.
#   - dropped when immediately combined via string::Concat/Format with a run-time value before
#     display (e.g. "  Gimbal " + Gimbal.Info() into a StringBuilder, "Requires: " + techName,
#     "Surv@" + FormatBurnTimeDisplay(...)) — confirmed by tracing the IL, since the text the UI
#     actually receives always differs from the bare literal.
# Also skipped: the #RF_* Localizer keys (RealFuels' own Localization/*.cfg owns those — this
# file only covers text hardcoded straight into IL), B9PartSwitch/TestFlightCore reflection
# type-name strings, *RFMEC*/[RF]/[MFS]/[MFT] Debug-log-only diagnostics and exceptions, ConfigNode
# parsing errors, ullage/gimbal ConfigNode key names, ModuleManager patch identifiers, and a
# module-name string compared with string::Equals (internal logic, never shown as text).
