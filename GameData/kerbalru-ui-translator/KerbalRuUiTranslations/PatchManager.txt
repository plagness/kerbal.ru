# PatchManager — отдельное меню для включения/выключения ModuleManager-патчей.
# Извлечено из #US-кучи (User Strings) DLL сканом UTF-16LE строк-литералов (`ldstr`).
#
# ВАЖНО: почти весь GUI этого мода (кнопки OK/Отмена/"Применить всё", диалог
# "Изменения вступят в силу...", настройки "Всегда показывать кнопку" и т.д.) уже
# переведён штатным способом — через ключи Localizer (`pm_ok`, `pm_applyall`,
# `pm_alwaysShow` и т.п.), определённые в самом моде:
#   GameData/PatchManager/Lang/en-us.cfg (английский оригинал)
#   GameData/PatchManager/Lang/ru.cfg    (готовый русский перевод, уже есть в моде)
# Сверка ключ-в-ключ подтвердила: en-us.cfg и ru.cfg содержат одинаковый набор
# pm_*-ключей — перевод полный. Эти строки резолвятся Localizer'ом ДО того, как
# попадают в GUI.Label/GUIContent, поэтому Harmony-перехватчику там ловить нечего
# (и дублировать их здесь не нужно и рискованно — легко разойтись в формулировках
# с уже вычитанным переводом мода).
#
# Ниже — то немногое, что реально зашито в коде как хардкод (не через Localizer):
# сообщения об ошибке неправильной установки мода, диалог сохранения и один
# тултип тумблера "Mod Enabled".
#
# Пропущено намеренно:
# - Все pm_*-строки (см. выше — уже переведены в Lang/ru.cfg).
# - "TitleNotAvailable"/"VersionNotAvailable" — служебные заглушки-сентинелы на
#   случай, если через reflection не удалось получить название/версию мода;
#   не связный UI-текст, переводить нечего.
# - "] Version " — фрагмент строки, склеиваемой в рантайме с именем мода и
#   номером версии (динамические данные), точного совпадения никогда не будет.
# - Debug.Log-сообщения ("PatchManager loaded configs, count: ", "Removing active
#   patch: ", "tryExit stopped/ended" и т.п.) — разработческая диагностика, не GUI.
