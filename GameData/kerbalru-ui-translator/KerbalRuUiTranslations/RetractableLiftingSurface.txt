# dialogName, title, message, buttonMessage, ...) -- a stock KSP call, only reachable if

# Everything else in the same call was checked and rejected, with a concrete reason each:
# - "test" (SpawnPopupDialog's `dialogName` arg) -- confirmed via Assembly-CSharp.dll's
#   MultiOptionDialog ctor (string name, string msg, string windowTitle, ...) that this is
#   the `name` field, used only for internal GameObject lookups (`<name> dialog handler`);
#   it is never routed through SetUpTextObject/TMP_Text and is not player-visible text.
# - The install-error body text ("Retractable Lifting Surface has been installed
#   incorrectly ... Incorrect path(s):\n") IS proven to reach SetUpTextObject/TMP_Text.text

#   string.Concat'd (in IL, before the SpawnPopupDialog call) with a runtime-built,
#   per-installation list of the actual wrong paths. The value TMP_Text.set_text receives
#   is therefore never the same twice and can never exactly match a fixed dictionary key --
#   same "concatenated with dynamic data, no exact match possible" situation already
#   documented for FShangarExtender/PatchManager's identical install-checker boilerplate.
#   Only the static title could be pulled out and translated; the message body cannot,
#   regardless of dictionary content.

#   same way but is only ever passed to UnityEngine.Debug.Log(object) -- console/log only,
#   never reaches any UI render call, so not in scope at all (same class of exclusion as
#   the AJE precedent: mod calls something, but never a patched entry point).
