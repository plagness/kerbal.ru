# Waterfall — UI strings extracted from Waterfall.dll (v0.11.0.0). Effect-editor mod, mostly
# used by advanced players/config authors. Two extraction passes:
#  1) `strings -a` on the raw DLL (identifiers + custom-attribute display names for the 15
#     controller types, e.g. [ControllerUIOptions("Gimbal")] style annotations -> UTF-8 attribute blob).
#  2) A custom UTF-16LE scanner (macOS `strings` cannot see .NET ldstr literals, which are UTF-16)
#     for the actual GUILayout.Label/Window text used throughout the in-game effect editor.
# Skipped by design: type/method/field identifiers, ConfigNode section/key names (CONTROLLER,
# EFFECTTEMPLATE, MODULE, engineID, etc. — these are .cfg schema keywords, not on-screen text),
# Debug.Log-only diagnostic/error messages ("[XController]: Could not find..."), shader/particle
# property ids (_MainTex, _Intensity...), .pdb paths, version/GUID/copyright metadata, and any
# string.Format template containing {0}-style placeholders (the substituted runtime text never
# equals the template verbatim, so such entries could never produce an exact match). Strings with
# an embedded literal newline are also skipped: the loader reads the file with File.ReadAllLines
# before it ever looks for the tab delimiter, so a dictionary line can't itself contain a newline.
