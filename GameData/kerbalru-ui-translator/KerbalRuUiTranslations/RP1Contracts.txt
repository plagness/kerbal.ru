# RP1Contracts -- словарь целей (objectives) контрактов RP-1, отрисовываемых стоковым
# приложением «Контракты» (KSP.UI.Screens.ContractsApp).
#
# МЕХАНИЗМ (подтверждён дизассемблировкой Assembly-CSharp.dll, monodis):
#   ContractsApp.CreateParameterList -> AddSubParameter(list, param, level) отрисовывает КАЖДЫЙ
#   параметр (в т.ч. верхнего уровня, level=0) так:
#       label = "    " + colors_levels[level] + param.Title + "</color>"
#       GenericCascadingList.CreateBody_spacer -> TMPro.TMP_Text.set_text(label)
#   т.е. до перехватываемой нами set_text доходит НЕ голый Title, а обёрнутая строка:
#       "    <color=#RRGGBB>ТЕКСТ</color>"
#   где цвет зависит ТОЛЬКО от уровня вложенности параметра:
#       level0=#ffffff level1=#bdbdbd level2=#7e7e7e level3=#636363  (ContractsApp .ctor)
#   Отступ "    " -- ровно 4 пробела (ldstr "    "), визуальный отступ по уровню задаётся
#   отдельно (LayoutElement.minWidth = 6*level), в строку НЕ входит.
#
# Поэтому ключи здесь -- это ТОЧНАЯ обёрнутая строка (4 пробела + цвет + текст + </color>),
# а не голый заголовок: голый заголовок до set_text не доходит и точным совпадением не ловится.
# Для каждого статического заголовка эмитим все 4 уровня-цвета (exact-match безвреден: лишние
# варианты просто никогда не совпадут). KSP 1.12.x заморожен -> значения цветов/отступа стабильны.
#
# Значение сохраняет ту же обёртку (отступ+цвет+</color>) вокруг русского текста, поэтому в игре
# сохраняются отступ, цвет и вложенность -- меняется только сам текст цели.
#
# ОХВАТ: только СТАТИЧЕСКИЕ заголовки параметров (без @-интерполяции) из
#   GameData/RP-1/Contracts/**/*.cfg  (PARAMETER { title = ... }). 448 уникальных.
# НЕ входят (точного совпадения не будет -- строка динамическая или генерится движком):
#   - заголовки с @ref (напр. "Maintain between @HoldSituation/minSpeed m/s ...");
#   - авто-заголовки Contract Configurator без литерального title= в cfg
#     (напр. "Vessel is built at SPH" от VesselBuiltAt) -- в cfg их нет, извлечь неоткуда;
#   - заголовки самих контрактов (CONTRACT_TYPE.title) -- идут другим путём отрисовки
#     и уже переведены в GameData/RP-1/Localization/kerbalru-quests-fix.cfg.
# Терминология согласована с kerbalru-quests-fix.cfg и docs/MAINTAINING.md.
#
# Формат: Английский<TAB>Русский. Несовпадение = строка остаётся как есть (см. tools/ui-translator).
# ФАЙЛ СГЕНЕРИРОВАН (tools/ui-translator, gen_dict.py) -- правь источник переводов, не этот файл вручную.
